# VirtualHair – Дипломен Проект (Master Plan)

VirtualHair е ASP.NET Core MVC уеб приложение, което позволява на потребителите да качат своя снимка, да пробват виртуално различни прически и бради чрез Canvas overlay, да запазват визии в лична галерия и да взаимодействат в мини социална среда. Целта е да се премахне несигурността при смяната на имиджа, предлагайки реалистична визуализация (напасване и цветове) преди посещение при фризьор.

## 1. Технологичен Стек
*   **Backend:** ASP.NET Core 9.0 MVC + Entity Framework Core (SQL Server / SQLite).
*   **Auth & Roles:** ASP.NET Core Identity (Individual Accounts) с роли Admin и User.
*   **Frontend:** HTML5, CSS3, Bootstrap 5.3, JavaScript.
*   **Взаимодействие & Визия:** HTML Canvas API, Cropper.js. Визуална идентичност "Liquid Obsidian" (тъмна тема, плавни анимации, модерен дизайн).
*   **Бъдещи интеграции:** `face-api.js` (AI разпознаване на лице за автоматично позициониране) и `Three.js` (мини 3D модел).

## 2. Архитектура и Домейни
Платформата се състои от следните основни модули:

*   **Identity & Roles:** Управление на регистрирани потребители. Админът управлява каталога, а потребителите само пробват и запазват визии.
*   **Catalog System:** CRUD операции за прически и бради (използват се локални PNG файлове с прозрачен фон в `wwwroot/uploads/`).
*   **Try Now Engine (Ядрото):** Изрязване на качена снимка с `Cropper.js`, налагане на минимум 2 слоя (Hairstyle, Facial Hair) върху Canvas, манипулация (Move, Scale, Rotate) и запазване.
*   **My Gallery System:** Потребителят записва визиите си със заглавие и може да ги преглежда.
*   **Profile System:** Управление на профила – промяна на потребителско име (уникално, чрез Identity без нужда от нови таблици/миграции), профилна снимка, парола.
*   **Community (Социална част):** Feed с визии, харесвания и коментари (В процес на преструктуриране за избягване на EF Core cascade проблеми).

## 3. Текущ Статус (Какво е направено)
✅ **Сигурност и Управление**
- Работеща Identity регистрация и вход.
- Ролите `Admin` и `User` се първоначално се зареждат през `IdentitySeeder`.
- Ограничени пътища за CRUD само за администратори.

✅ **Каталози и База Данни**
- Изградени домейни `Hairstyle`, `FacialHair`, `UserHairstyle`.
- Качване на картинки и запис на физическите им пътища (`ImagePath`).
- Отстранени стари EF Core проблеми (Multiple Cascade Paths са изолирани чрез `NoAction`/`Restrict`).

✅ **UI/UX и Профил**
- Custom Offcanvas навигация и "Dark/Light" режим с `localStorage` persist.
- Loader / Shimmer ефект при зареждане на страници.
- Профилната страница поддържа качване на снимка с изрязване, смяна на съществуващото `UserName` и интеграция за смяна на паролата. 

✅ **Try Now и Gallery**
- Изскачащ прозорец за кропване на оригиналната снимка до квадрат (320x320 px).
- Рендериране на Canvas с базов контрол за нагласяне на коса/брада.
- Запазването (`Save Look`) генерира Base64, превръща се във файл и се записва в базата като отделен поглед.

## 4. План на действие (Roadmap)

**🔥 Фаза 1: Стабилизация на Try Now (Основен приоритет СЕГА)**
1. **Fix Layer Reset:** При избиране на нов слой/смяна на прическа да не се губи старото състояние (zoom/position) освен ако не е умишлено нужно.
2. **Save Look 100%:** Уверявка, че записът на визия работи перфектно (генерация на файл и база).
3. **Active Layer Selection:** Клик върху Canvas елемент (коса или брада) автоматично да го прави активен слой за манипулация.
4. **Color Picker (Canvas Tint):** Възможност за базово оцветяване на прическата чрез canvas blend режими (напр. `source-atop`).

**🟡 Фаза 2: Изчистване на профила (Profile)**
1. Редакция на `UserName` (Направено).
2. Запис на профилна снимка след кропване (В процес).
3. Разкрасяване на UI/търсене на малки дефекти.

**🔵 Фаза 3: AI Auto Align (WOW Елемент)**
1. Интеграция на `face-api.js` в `preview.js`.
2. Засичане на `landmarks` (очи, нос, челюст).
3. Автоматично изчисление на Offset (X, Y), Мащаб (Scale) и Ротация (Rotation) за прическата спрямо главата.
4. Ако AI не намери лице – fallback към default центрирана позиция и ръчни контроли.

**🟢 Фаза 4: Social Feed (Чисто пренаписване)**
1. Възстановяване на `Post`, `Like`, `Comment` моделите без `ambiguity` и каскадни грешки.
2. 1 Like за 1 потребител на 1 пост (Уникални ограничения).
3. Минималистичен и елегантен Feed UI.

**🟣 Фаза 5: Extras (При наличие на време)**
- Slider "Before/After" за лесно преглеждане на резултат.
- История "Undo/Redo" на Canvas трансормации.
- 3D Head Demo (Three.js).
- Мини-игри или Messaging / Friends модул.

## 5. Правила на проекта
За да не се "счупи" стабилността отново, спазваме следните правила:
1. ❌ Няма нови бази данни и миграции без ясна и конкретна необходимост.
2. ❌ Няма каскадни `delete` по подразбиране, за да се избегнат multiple cascade path грешки.
3. ✔️ Една задача се започва и се довършва напълно, преди да се премине към следващата.
4. ✔️ Core Engine-ът (Try Now) е най-важен. Всичко останало се гради около него.

## 6. Как да стартирате проекта
### Изисквания:
* Инсталиран [.NET SDK 9.0](https://dotnet.microsoft.com/download)
* SQL Server (вижте `appsettings.json` за Connection String-а)
* Visual Studio 2022 (препоръчително)

### Стъпки:
1. Отворете `VirtualHair.sln` във Visual Studio 2022.
2. Отворете `Package Manager Console` (Tools -> NuGet Package Manager) и изпълнете: `Update-Database`
3. Натиснете **F5**, за да стартирате.
4. (Опционално през терминал) Отворете папката и изпълнете `dotnet ef database update`, след което `dotnet watch run`.
